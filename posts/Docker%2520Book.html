Docker 基础与实战入门笔记 ​

Docker 用容器打包应用及其运行环境，让同一个应用能以一致的方式在本地、测试和服务器上运行。 Docker是一个用于 构建 运行 传送 应用程序的平台 可以将操作系统（OS），运行时环境，第三方软件库和依赖包，应用程序，环境变量，配置文件，启动命令等打包在一起 ，以便在任何环境中都可以运行

  • 理解：镜像比喻成菜单，容器比喻成菜品。我们给别人的是镜像，也就是菜单；别人用菜单去弄出一道同样的菜，菜就是容器

1. 核心概念 ​

名称含义类比
镜像（Image）只读的应用运行模板安装包或系统模板
容器（Container）镜像运行后的实例正在运行的应用进程
Dockerfile构建镜像的步骤文件镜像的制作说明书
数据卷（Volume）容器外部的持久化数据独立的数据磁盘
镜像仓库（Registry）存储和分发镜像的位置Maven、npm 仓库
Docker Compose用 YAML 管理多个容器多服务启动清单
text
Dockerfile -- docker build --> 镜像 -- docker run --> 容器

容器不是虚拟机。多个容器共享宿主机内核，启动更快、占用资源更少；但容器内未挂载的数据，在删除容器后通常会丢失。

2. 安装后检查 ​

bash
docker --version
docker compose version
docker info
docker run hello-world

Linux 用户不想每次输入 sudo 时，可以将用户加入 docker 组：

bash
sudo usermod -aG docker $USER

重新登录终端后生效。注意：docker 组拥有接近 root 的权限，不应随意添加不可信用户。

3. 镜像常用操作 ​

bash
docker images                         # 查看本地镜像
docker search nginx                   # 搜索 Docker Hub 镜像
docker pull nginx:1.27-alpine         # 拉取指定标签镜像
docker image inspect nginx:1.27-alpine # 查看镜像详情
docker rmi 镜像ID                      # 删除本地镜像
docker image prune                    # 清理无悬挂镜像

镜像名通常为：

text
仓库地址/命名空间/镜像名:标签

例如 mysql:8.4 和 nginx:1.27-alpine。生产环境建议固定具体版本，避免使用 latest。

4. 容器常用操作 ​

  • 容器化和Dockkerfile

容器化：将应用程序打包成容器，然后在容器中运行应用程序的过程

step1：创建一个Dockerfile

step2：使用Dockerfile构建镜像

step3：使用镜像创建和运行容器

4.1 启动一个容器 ​

bash
# 前台运行，Ctrl + C 停止
docker run --rm nginx:1.27-alpine

# 后台运行，映射主机 8080 到容器 80 端口
docker run -d --name web -p 8080:80 nginx:1.27-alpine

# 查看容器
docker ps
docker ps -a

端口映射格式为 主机端口:容器端口。访问 http://服务器IP:8080 时，流量会进入容器的 80 端口。

4.2 管理容器 ​

bash
docker start web
docker stop web
docker restart web
docker rm web
docker rm -f web                  # 强制停止并删除
docker logs web
docker logs -f --tail 100 web     # 持续查看最近 100 行日志
docker exec -it web sh            # 进入容器 Shell
docker inspect web                # 查看网络、挂载和配置

docker stop 会先尝试优雅停止，超时后才强制结束。不要依赖进入容器手动修改文件，容器重建后这些改动会消失。

5. 环境变量与数据卷 ​

5.1 传递环境变量 ​

bash
docker run -d \
  --name mysql \
  -e MYSQL_ROOT_PASSWORD=change-me \
  -e MYSQL_DATABASE=demo \
  -p 3306:3306 \
  mysql:8.4

不要把真实密码直接写进 Git 仓库。开发环境可使用本地 .env 文件，并将其加入 .gitignore。

5.2 数据持久化 ​

命名数据卷：

bash
docker volume create mysql-data

docker run -d \
  --name mysql \
  -e MYSQL_ROOT_PASSWORD=change-me \
  -v mysql-data:/var/lib/mysql \
  mysql:8.4

绑定宿主机目录：

bash
docker run -d \
  --name web \
  -p 8080:80 \
  -v /opt/web/dist:/usr/share/nginx/html:ro \
  nginx:1.27-alpine

其中 :ro 表示只读挂载。常用查看与清理命令：

bash
docker volume ls
docker volume inspect mysql-data
docker volume prune

docker volume prune 会删除未被使用的数据卷，执行前先确认没有需要保留的数据。

6. 容器网络 ​

Docker Compose 会自动为同一项目创建网络。服务之间应通过服务名通信，而非依赖固定容器 IP。

bash
docker network ls
docker network create app-network
docker network inspect app-network

手动创建两个可通信的容器：

bash
docker run -d --name redis --network app-network redis:7-alpine
docker run -it --rm --network app-network redis:7-alpine redis-cli -h redis ping

第二条命令中 redis 是容器名，同时也可作为网络内的主机名。

7. Dockerfile 构建镜像 ​

以下示例将一个 Spring Boot JAR 打成镜像：

dockerfile
FROM eclipse-temurin:21-jre-alpine

WORKDIR /app

COPY target/app.jar app.jar

EXPOSE 8080

ENTRYPOINT ["java", "-jar", "app.jar"]

构建和运行：

bash
docker build -t myapp:1.0.0 .
docker run -d --name myapp -p 8080:8080 myapp:1.0.0

常用 Dockerfile 指令：

指令作用
FROM指定基础镜像
WORKDIR设置工作目录
COPY将构建上下文中的文件复制进镜像
RUN构建镜像时执行命令
ENV设置环境变量
EXPOSE声明容器使用的端口
CMD / ENTRYPOINT指定容器启动命令

建议添加 .dockerignore，避免把无关文件发到构建上下文：

gitignore
.git
.idea
node_modules
target
*.log
.env

如果 Dockerfile 依赖 target/app.jar，不要在 .dockerignore 中忽略它；应在构建镜像前先执行项目打包命令。

8. Docker Compose 管理多服务 ​

Docker Compose 用一个 compose.yaml 启动应用、数据库等多个服务。

yaml
services:
  app:
    image: myapp:1.0.0
    container_name: demo-app
    ports:
      - "8080:8080"
    environment:
      SPRING_PROFILES_ACTIVE: prod
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/demo
      SPRING_DATASOURCE_USERNAME: root
      SPRING_DATASOURCE_PASSWORD: change-me
    depends_on:
      mysql:
        condition: service_healthy
    restart: unless-stopped

  mysql:
    image: mysql:8.4
    container_name: demo-mysql
    environment:
      MYSQL_ROOT_PASSWORD: change-me
      MYSQL_DATABASE: demo
    volumes:
      - mysql-data:/var/lib/mysql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost", "-pchange-me"]
      interval: 10s
      timeout: 5s
      retries: 5
    restart: unless-stopped

volumes:
  mysql-data:

常用 Compose 命令：

bash
docker compose up -d
docker compose ps
docker compose logs -f
docker compose logs -f app
docker compose exec app sh
docker compose down
docker compose down -v      # 同时删除数据卷，谨慎使用
docker compose pull
docker compose up -d --build

9. 一个前端 + Nginx 示例 ​

将前端构建产物部署到 Nginx：

dockerfile
FROM nginx:1.27-alpine

COPY dist/ /usr/share/nginx/html/
COPY nginx.conf /etc/nginx/conf.d/default.conf
nginx
server {
    listen 80;
    server_name _;

    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://app:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}

这里的 app 应是 Compose 中后端服务的名称；Docker 内部网络会解析这个名称。

10. 排错与清理 ​

排错常用命令：

bash
docker ps -a
docker logs 容器名
docker inspect 容器名
docker stats
docker system df
docker events

清理无用资源：

bash
docker container prune
docker image prune
docker network prune
docker system prune
docker system prune -a

docker system prune -a 会删除未被容器使用的镜像，之后重新启动服务可能需要重新拉取或构建镜像。生产服务器执行清理前应先查看 docker system df。

11. 日常部署流程 ​

bash
# 1. 拉取最新代码
git pull

# 2. 构建或拉取新镜像
docker compose build
# 或 docker compose pull

# 3. 后台重建服务
docker compose up -d

# 4. 查看状态与日志
docker compose ps
docker compose logs -f --tail 100

12.具体深入学习 ​

Docker 的重点是把“应用如何运行”写成可重复执行的配置。镜像负责交付运行环境，数据卷负责保留数据，Compose 负责管理多个相互依赖的服务。

这是重点！ ​
